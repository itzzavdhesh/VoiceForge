# Security Policy

VoiceForge handles voice samples, generated speech, webcam previews, and API keys. Please do not open public issues for security vulnerabilities or privacy bypasses.

## Reporting A Vulnerability

Email the maintainer or use GitHub private vulnerability reporting if it is enabled for this repository.

Include:

- A clear description of the issue.
- Steps to reproduce.
- Impact and affected files or endpoints.
- Any proof of concept, logs, or screenshots that are safe to share privately.

## Security Expectations

- Do not submit real API keys, access tokens, private voice samples, or personal recordings.
- Do not test with another person's voice unless you have explicit consent.
- Do not use issues or PRs to publish exploit details before a fix is available.
- Keep dependency updates small and reviewable.

## Supported Versions

Only the latest `main` branch is currently supported.
